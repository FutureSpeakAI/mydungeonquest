# MyDungeon.Quest

An AI-narrated solo tabletop RPG: a Dungeon Master model runs a persistent, locally-stored fantasy campaign for one player, complete with dice rolls, combat, cinematics, and generated art/audio.

## Run & Operate

- `pnpm --filter @workspace/mydungeon-quest run dev` — run the game (Vite dev server + Express API, both on the artifact's routed port)
- `pnpm --filter @workspace/mydungeon-quest run build` — production build (outputs to `artifacts/mydungeon-quest/dist`)
- `pnpm --filter @workspace/mydungeon-quest run eval` — headless bench: validates DM protocol, reducers, canon integrity, PG-13 scrubber, budget caps, seal/tamper invariants. Keep this green.
- `pnpm --filter @workspace/api-server run dev` — run the (separate, unrelated) API server template artifact
- Required secrets: `ANTHROPIC_API_KEY` (real AI Dungeon Master; falls back to a deterministic mock without it), `OPENAI_API_KEY` (real illustration art via the Paint provider), `ELEVENLABS_API_KEY` (voice/music/SFX)
- The door (optional accounts): `CLERK_SECRET_KEY` + `CLERK_PUBLISHABLE_KEY` (server) and `VITE_CLERK_PUBLISHABLE_KEY` (client build) raise the door — absent, there is no sign-in UI and everyone is a guest. `DATABASE_URL` backs the patron ledger; the `users` table self-bootstraps (`CREATE TABLE IF NOT EXISTS`, in `server/patrons.js`) on first inscription, so fresh environments need no hand-run migration. If the ledger or Clerk is unreachable, signed-in players degrade to guests (fail-open, loud `[door]`/`[ledger]` server logs) — deliberate for the tale itself; the toll-house (see Architecture decisions) fails CLOSED for paid pours when the ledger of use cannot be read. Stripe arrives via the Replit Stripe connector (no secret to set by hand); the gateway is live only when door AND mint both stand.
- Provider selection env vars (shared, non-secret): `PAINT_PROVIDER=openai`, `SPEAK_PROVIDER=elevenlabs`, `MUSIC_PROVIDER=elevenlabs`, `SFX_PROVIDER=elevenlabs`, `DM_MODEL`/`DM_MODEL_GENESIS` (Anthropic model ids)

## Stack

- React 19 + Vite 7, plain JS/JSX (not TypeScript by design)
- Express 5 server (`artifacts/mydungeon-quest/server/`) serving `/api/*` (DM turns, image/voice/music/sfx generation, PDF character sheet binding via Playwright) and, in production, the built static app
- Client-side persistence via Dexie/IndexedDB — chronicles are fully local-first; the only Postgres table is the account ledger (`users`: Clerk id + display name, never tales)
- Anthropic Claude (tool-calling) for the Dungeon Master; OpenAI for illustrations; ElevenLabs for voice/music/SFX

## Where things live

- `artifacts/mydungeon-quest/` — the whole game (imported wholesale from an external build, not scaffolded from a template)
  - `src/` — React app; `server/` — Express API + adapters; `scripts/dev.mjs` — dev process orchestration; `evals/` — headless bench suite
  - `GAME_NOTES.md`, `BUILD_STATUS.md`, `CHANGELOG.md`, `NOTICE.md`, `README.md` — the game's own original docs, preserved (lightly corrected after the July 2026 film retirement); the repo-level `README.md` at the workspace root is the canonical public documentation

## Architecture decisions

- This artifact was imported wholesale from a finished external build rather than adapted to the repo's usual OpenAPI/shared-backend/Postgres conventions — it intentionally keeps its own self-contained Express server and IndexedDB storage.
- Dev mode runs two processes: Vite owns the artifact's single externally-routed port; the game's own Express server runs on an internal-only port (`INTERNAL_API_PORT`, default 3001) and is reached via Vite's `/api` proxy. Production instead builds static assets and lets Express serve everything on the routed port directly.
- Added `@vitejs/plugin-react` (absent from the original project) because Vite 7's default esbuild JSX transform is not "automatic" without it, which broke JSX at runtime ("React is not defined").
- Extended the Anthropic tool JSON schema in `server/dm.js` with the same enums the client-side validator (`src/lib/protocol.js`) already enforces (roll kinds, dice, advantage, cinematic types, combat ops, zones, image cue kinds, time units). Without these, newer Claude models produced structurally valid but semantically out-of-range values that the strict validator rejected, silently falling back to mock narration on every turn.
- The Quieting (July 2026): every non-narration sound flows through a single client Audio Director (`src/lib/cinema/audioDirector.js`) enforcing THE SOUND LAW — one voice at a time, music/SFX only as short punctuation phrases (never beds, never loops), voice preempts music instantly, late accents are dropped, and mock-provenance audio is refused at delivery (keyless tables are silent). The WebAudio drone score (`score.js`) and speechSynthesis dialogue (`voice.js`) were deleted; the `score`/`voice` settings toggles were removed and legacy values are stripped on settings load.
- The Cast Made Flesh (July 2026): voices are cast by reading the card once at first introduction and persisted on the cast card (`voiceId`); pre-law souls keep their legacy name-hash voice (never recast mid-tale). Cast cards carry `known_facts[]`, `bond_arc`, `introduced_turn`, and a status enum (`active|dead|missing`) with resurrection retcons refused. One dm_turn law amendment — THE DEAD DO NOT SPEAK — is enforced by threading the pre-turn cast snapshot into `validateDmTurn` on the client turn path and every server validate/repair loop (the context arg is optional, so new call sites must remember to thread it).
- Acts, Chapters, and Endings (July 2026): the story reducer (`src/lib/story.js`) owns endings — arriving at the final beat no longer completes the tale (the epilogue beat must be *played*; advancing while already on it completes), and "Seal the Tale" records `codex.sealing` which steers the DM into a short combat-free denouement via [STORY] directives, completing at `final_turn` even across `story:null` turns. The UI chains act interstitials through the same Cinematic card (DM card → act card → narration, per THE SOUND LAW), retires the composer on completion, and the Sealing ceremony appends one final `sealing` journal event (turn/roll counts) before marking `campaign.sealedAt`. The mast reads `chapterInfo`/`actInfo`.
- The Chronicler (July 2026): a second, smaller harness (`/api/retell`, one forced `chronicle_passage` tool call, `server/retell.js`) retells each closed chapter from its sealed turns under three laws enforced by the shared validator in `src/lib/chronicler.js` — it may not invent (citations + codex-name checks), may not contradict (sealed dice totals only; the dead are not quoted after the turn they fell), it may only retell (declared verbatim quotes with edge-punctuation forgiveness; digits stay out of the prose). The client re-judges every wire response, seals lawful pages as written (`chronicle_page` journal events, mirrored on `campaign.chroniclePages`), and hangs them in the feed (`src/components/ChroniclePage.jsx`) after the turn that closed the chapter. Keyless or twice-unlawful → honest decline: **no mock prose is ever sealed**; the storybook binds raw sealed text instead (`rawChroniclePassage`, consumed in Phase 4). Redacted turns never reach the Chronicler — not their words, not their deaths. Evals: `evals/chronicler.test.mjs` (three laws + schema/validator lockstep).
- The Storybook, second binding (July 2026): `buildStorybook` v2 compiles sealed journal + `campaign.chroniclePages` + painted plates into one illuminated book with three lives — an in-app page-turn reader (self-contained script; stands down under `navigator.webdriver` so `/api/bind-pdf` prints clean), Letter/A5 print CSS (folio toggle in the Storybook overlay rebinds via `openStorybook(size)`), and a standalone keepsake carrying an embedded verifier-format proof (`mydungeon.chronicle` JSON) the notary at `/verify.html` accepts. Chapters bind the Chronicler's sealed retelling (drop cap, dice marginalia, entrance portraits) or honestly fall back to the raw sealed record; redaction is journal law first (redaction events strike turn ordinals; the log flag is the belt); every image passes a strict `data:image/*;base64` whitelist (crafted MIMEs from imported chronicles are refused) and the overlay iframe is sandboxed; keyless builds succeed with procedural plates. Evals: `evals/storybook.test.mjs`.
- The Podcast Forge (July 2026): one produced episode from the sealed record. `src/lib/podcast.js` compiles the script (cold open — the highest-stakes sealed quote in its own character's voice → title liturgy → per-beat sections: Chronicler passages read whole + cast re-speaking sealed lines; raw sealed narration when no page exists → "Sealed, and true.") and `validatePodcastScript` is the verbatim court — every segment must match its cited source exactly; journal-law redaction strikes here too. The mix is a sequencer, not a mixer: `buildMixPlan`/`assertLawfulPlan` prove voice/gap/sting sequences (stings only BETWEEN sections), and `server/sequencer.js` renders ONE ffmpeg concat chain — loudnorm −16 voices/−19 stings, generated-silence gaps, chapter markers on the true probed timeline, key-art cover through a strict data-URI door. The music bed is retired: `bed` → 400 by name, mock provenance → 422, keyless decline: "The podcast needs voices. Add a key, or keep the book." Evals: `evals/podcastForge.test.mjs`.
- The Door (July 2026, SaaS phase 1): player accounts via Replit-managed Clerk, OPT-IN like every media provider — no keys, no door. Client (`src/patron/door.jsx`): `PatronShell` wraps the app in `main.jsx` only when `VITE_CLERK_PUBLISHABLE_KEY` exists (wouter routes `/sign-in/*?` + `/sign-up/*?`, themed ClerkProvider, canonical `publishableKeyFromHost` + unconditional `proxyUrl`); keyless builds render the game byte-for-byte unchanged (no router, no provider — the eval harness pins the key to `''` in `evals/jsxLoader.mjs`). `PatronDoor` is a quiet line inside `.title-footer`, riding the existing arrival/attract fades. Server: `server/clerkProxy.js` (prod-only FAPI proxy at `/api/__clerk`, mounted before body parsers) and `server/patrons.js` — the doorkeeper on `/api` reads the Clerk session, inscribes patrons into Postgres `users` (upsert by `clerk_user_id`, promise-cached per process, name fetch raced against a 3s candle), and marks everyone else a guest; failures fail OPEN to guest while no privileged routes exist. `GET /api/whoami` answers honestly. Chronicles stay on-device; the ledger holds names, never tales. Evals: `evals/doorkeeper.test.mjs`.
- The Toll-House (July 2026, SaaS phase 2): subscriptions & the metered gateway, OPT-IN twice over — it stands only when both the door (Clerk) and the mint (Stripe, via the Replit connector) are live; anything less is a dormant hallway (unmetered, byte-for-byte legacy behavior). `server/mint.js` builds the Stripe client from the connector (fetch ALL connections unfiltered and pick `connector_name === 'stripe'` — the `connector_names` query filter returns nothing; the secret lives at `settings.secret`) and runs `StripeSync` into the `stripe` Postgres schema (`syncBackfill({ object: 'all' })` — a bare call syncs nothing — plus a managed webhook). `server/toll.js` holds the law: PLANS/quotas (guest/free — the lifetime taste/weekly $5/yearly $129.99, plus `house` "Friend of the house": a comp seat written into `users.plan` by the owner's hand — `comp: true`, unmetered, never sold, never on the price board, and reconciliation leaves it untouched without consulting Stripe; `null` = unmetered, `0` = not on the table; calendar-month UTC), the innkeeper middleware (402s always in the house's own words: `{closed:true, reason:'table'|'spent'|'mislaid', upsell, error}`), the `usage_events` ledger (debit AFTER work; mock/fallback stand-ins never billed; podcast/pdf bill as 'house'; fail closed for money, open for the table), `reconcileEntitlement` (re-reads `subscriptions.list`, `price.metadata.mdq_plan`, highest active/trialing/past_due seat wins — flips `users.plan` both directions, except `comp` seats which it returns untouched without consulting Stripe: the mint neither grants nor revokes a gift, and a comp mark chalked on a price raises no seat), the price board off `stripe.prices`, and checkout/portal/refresh routes (raw-body webhook mounted before parsers). Client `src/patron/toll.jsx`: `useToll`/`TollSection` in Settings (standing, tallies, seat upgrades, portal), `tollAllows` advisory pre-filter on media jobs, `settleTollReturn` for `?toll=paid|kept|seen` returns. Seats are chalked idempotently by `scripts/src/seed-toll.ts` (`metadata.mdq_plan` on product AND price). Evals: `evals/tollhouse.test.mjs`.
- The Watchtower & the Front Gate (July 2026, SaaS phase 3 — production launch): `server/watchtower.js` hardens the house for the public. Structured JSON logs (one line per /api request; `LOG_JSON=1` or production; never bodies/tales), process tripwires (`installAlarms` — unhandled rejections/uncaught exceptions ring `level:alert` lines and are tallied on `/api/health`), DURABLE burst limits (the rate window counts in Postgres `rate_windows` when `DATABASE_URL` stands so restarts don't reset it; in-memory belt otherwise and on ledger stumbles — never a 500), abuse caps (`abuseCaps(kind)` — 413s on oversized turn bodies/prompts/plate dimensions, env-tunable `MAX_PROMPT_BYTES` etc.), and per-provider DAILY SPEND CEILINGS (`spend_days` table + in-process mirror, higher count wins; `SPEND_CEILING_<PROVIDER>`/`SPEND_CEILING_DEFAULT`, default 2000/day; a spent provider is skipped in the media chain and the DM degrades to mock via `getDmTurn`'s `forceMock` — graceful, never dark, stand-ins never billed). `/api/health` is the uptime surface (door, ledger pulse, alarms, spend tallies). THE HERALD (opt-in): set `ALERT_WEBHOOK_URL` (Slack/Discord/any JSON-POST webhook) and alarms + first daily ceiling-strikes ring out as notifications — throttled to one per alarm kind per hour and one per provider-ceiling per UTC day; unconfigured forks send nothing and log nothing new; a failed delivery logs `herald_undelivered`, never throws. The front gate is static brand-voice pages in `public/`: `welcome.html` (landing + pricing tiers, one-touch guest play), `terms.html`, `privacy.html` — linked from the title footer (`.law-links`), the toll seats block (`.toll-legal`), and each other. PDF binding resolves Chromium from `PLAYWRIGHT_CHROMIUM_PATH` → `/usr/bin/chromium` → PATH (the Nix `chromium` system package is installed, so deployments carry the press). Deployment: artifact.toml production config builds Vite then runs `node artifacts/mydungeon-quest/server/index.js` with `NODE_ENV=production`; ffmpeg and chromium ride the Nix environment. Evals: `evals/watchtower.test.mjs`.
- Drama & Brand (July 2026): the experience cut. The Arrival — a once-per-sitting cold open (dark → candle → title → shelf; any touch skips it; `sessionStorage mdq:arrived`) ending on the tagline "A tale that is actually yours."; the shelf is a library of book spines (deterministic dye by campaign id, height grows with turns told, wax sigil once sealed) and a sealed spine opens straight to its keepsakes; combat pulls a vignette over the whole shell (`.under-arms`); the dice ritual stages a spotlight and parchment disc — gold for success, ash for failure, criticals hold one extra breath (`RITUAL_MS`/`CRIT_HOLD_MS` in `DiceOverlay.jsx`); plates are numbered as a folio ("Plate IV · illuminated"); `verify.html` is the Notary's Desk; UI sfx remain single deliberate accents through the Director (first touch = one page-turn), so the keyless first-run is full drama, total silence. Evals: `evals/arrival.test.mjs`.
- The Android Shell (July 2026): `artifacts/mobile` wraps the live site in an Expo WebView glass. THE DOOR LAW (`lib/door.js`, pure and benched by `lib/door.test.mjs` via `pnpm --filter @workspace/mobile run check`) keeps house hostnames in-glass and resolves every doubtful parse OUTWARD to the system browser — WHATWG-aligned manual parsing (backslash ends the authority, exactly as Chromium reads it), userinfo/case/trailing-dot tricks all fall out, script/data pseudo-URLs drop cold. Android back walks WebView history and asks only at the true threshold; loading is a breathing Rose, offline is the Cold Hearth ("Stir the fire") — never a spinner. The web side carries the store dowry: hardened `manifest.webmanifest` (id/scope/orientation + maskable PNG icons in `public/icons/`, rendered by `scripts/render-icons.mjs`), an apple-touch-icon, and `/.well-known/assetlinks.json` from `server/dowry.js` — env-chalked signing fingerprints, honest-empty when none are set — proxied in dev via Vite so preview answers like production. Evals: `evals/dowry.test.mjs`.
- The Living World Cut (July 2026, Directive IV — lifted from the outside forge's drop): THE TENOR LAW — `cast_add`/`cast_update` carry an optional `voice` card (gender/age/timbre), the hero states presentation and pronouns at the forge, and casting (`src/lib/cinema/casting.js`) reads stated identity through a typed voice roster before any lexicon prior (applied to name+role words only, never canon prose); the prose coin-flip is retired, cast `voiceId`s are grandfathered, the dead still do not speak. THE CARD LAW — `src/lib/cards.js` is a pure reducer replaying validated ops into one Character Card per soul, hero included (locked identity, ops-only state, cited chronicle, typed ties; byte-identical on replay). THE CHRONICLEGRAPH — `src/lib/graph.js` rebuilds [STORY] as a budgeted scene-first context pack (souls in scene, one-hop ties, the villain always; `storyBlock` contract preserved as fallback in App). THE LIVING WIKI — Codex soul pages render cards as story (voice in words via `src/lib/wikiText.js`, tie-chip backlinks, appearances timeline; the book's dramatis personae quotes first/last sealed words). THE LIVING WORLD — `src/lib/livingWorld.js` advances up to four goal-bearing souls one bounded, deterministic, ops-only stride on `time_advance` or an act turn, sealed as `tick` journal entries (skipped by DM history and the book). THE TWO-SPEED FORGE — three doors each for world and hero (Spin the World / Oracle / Deep Forge; Cast the Bones / Oracle / Forge by Hand), per-field dice via `src/lib/forgeRolls.js`, voice audition through `/api/speak`. Six new append-only gates: `tenor`, `cards`, `graph`, `wiki`, `livingWorld`, `forge`. Merge note: the drop's snapshot predated the Android-shell merge, so its collisions on `server/index.js`, `vite.config.js`, `index.html`, `manifest.webmanifest`, and `package.json` were staleness, not intentions — our dowry/manifest/proxy/SW-destructor work kept, their six gates appended to our eval chain, our lockfile kept (no dependency changes).
- The Sequencing Cut (July 2026, v0.5.1 — lifted from the outside forge's drop, built to the amended Task #50 plan): presentation law over the sealed record — nothing here writes a log or journal entry, and no tick is touched. Sealed `tick` entries render as time-passes dividers (`src/components/Sequence.jsx`: the phrase reads the sealed neighbor turn's `time_advance`, up to two whispers quote the tick's own ops) instead of the empty turn rows they silently were. The feed's seat order is a pure, gated function (`orderFeed` in `src/lib/sequencing.js`): closing turn → chronicle page → tick divider → next chapter; a page whose anchor is gone or struck recovers at its beat's true boundary (never lost to arrival timing), and a "The Chronicler is writing…" pending row holds only a seat a page can still take — keyless/doorless tables hold none by construction (the toll never fetches there). Re-entry staging (`recapFor` + `RecapCard`): "The tale so far" from sealed material only — latest lawful chronicle page, mast-only fallback (act · chapter · goal), honest silence for fresh or finished tales; struck pages and struck anchors never shown; once per sitting, in memory only. Suggestion chips stagger in after a short reading beat (instant under reduced motion; typing and voice never gated). Gate: `evals/sequencing.test.mjs`, appended to the chain. Merge note: the true delta was isolated by diffing this drop against the prior drop's tree (seven lifted files + the gate append); their base again predated our dowry/PWA work and the Overlays.jsx import fix — all kept ours. ## Product

- Forge a new world (campaign), play through DM-narrated turns with dice rolls and combat, receive cinematic story beats and generated illustrations, and export/print a character sheet as PDF. Chronicles are stored locally in the browser; accounts are optional — patrons may give their name at the door (Clerk) and the ledger keeps names, never tales.

## Gotchas

- Real Anthropic/OpenAI/ElevenLabs calls require the corresponding secret to be set — without it, the game gracefully falls back to mock providers rather than failing.
- PDF character sheet export (`/api/bind-pdf`) needs a Playwright Chromium binary (`npx playwright install chromium` inside `artifacts/mydungeon-quest`).
- Anthropic model ids change frequently; if the DM starts silently falling back to mock/fallback narration, check `DM_MODEL`/`DM_MODEL_GENESIS` against `GET /v1/models` for currently valid ids.
